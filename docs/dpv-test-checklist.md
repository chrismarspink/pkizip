# DPV 통합 검증 체크리스트

PKIZIP 의 DPV (Phase 1·2) + 가/익명화 실제 적용 동작 검증을 위한 수동 테스트 가이드.

각 시나리오 끝의 ✅ 항목을 모두 통과해야 운영 배포 가능.

---

## 0. 사전 준비

```bash
cd /Users/chris/HE/pkizip
npx vite          # dev 서버 실행
```

테스트 파일 준비 (각 포맷 1개씩, PII 1~3개씩 포함):
- `test_email.txt` — 평문, 이메일·전화번호
- `test_account.csv` — CSV, 주민번호·계좌번호
- `test_payroll.xlsx` — 셀에 주민번호·이메일
- `test_meeting.pptx` — 슬라이드에 이름·전화
- `test_proposal.docx` — 문장에 여권번호
- `test_report.hwpx` — section 에 주민번호
- `test_legacy.hwp` — HWP 바이너리 (있다면)
- `test_contract.pdf` — PDF 텍스트, 이름·주민번호
- `test_namecard.jpg` — 명함 사진 (OCR 대상)

## 1. Phase 1 검증 — 데이터 카테고리 메타 자동 부착

### 시나리오 1.1: 평문 텍스트 봉투 생성
**입력**: `test_email.txt` 에 `사용자: hong@example.com 010-1234-5678`

**절차**:
1. CreatePage → 파일 추가
2. "다음" → 분석 다이얼로그 자동 진입
3. 가명화는 "건너뛰기" 선택
4. CMS 옵션 그대로 → 생성

**확인 항목**:
- [ ] ✅ 분석 다이얼로그에 보라색 "DPV 표준 분류" 섹션 표시
- [ ] ✅ `dpv:EmailAddress`, `dpv:TelephoneNumber` 포함
- [ ] ✅ 한국어 라벨 "이메일", "전화번호" 표시
- [ ] ✅ 봉투 생성 후 ExplorerPage 에서 카드에 칩 보임
- [ ] ✅ 카드 hover 시 tooltip 에 IRI 목록
- [ ] ✅ 봉투 상세 패널에 "DPV 표준 메타" 섹션

### 시나리오 1.2: PII 미검출 파일
**입력**: 일반 텍스트 (`회의록.txt`) — PII 없음

**확인 항목**:
- [ ] ✅ 봉투 헤더에 `dpv` 필드 자체가 없음 (또는 빈 카테고리)
- [ ] ✅ ExplorerPage 카드에 DPV 칩 안 보임
- [ ] ✅ 등급 O

## 2. Phase 2 검증 — 처리 활동 + 적용 조치 자동 부착

### 시나리오 2.1: 외부 전송 + 암호화 + 서명 + TSA
**입력**: `test_payroll.xlsx` (주민번호 포함)

**절차**:
1. 분석 → 가명화 "건너뛰기" → 의도: "외부" 선택 → Hybrid 암호 모드
2. 봉투 생성

**확인 항목**:
- [ ] ✅ 봉투 상세 패널 "DPV 표준 메타" 안:
  - [ ] **데이터 카테고리**: `dpv:NationalIdentifier`
  - [ ] **처리 활동** (파란색): `dpv:Storage`, `dpv:Transfer`, `dpv:Encrypt`
  - [ ] **적용 조치** (녹색): `dpv:Encryption`, `dpv:DigitalSignature`, `dpv:TimestampingService`, `dpv:CryptographicMethods` (PQC)

### 시나리오 2.2: 내부 보관만
**입력**: 같은 파일, 의도 "내부" 선택

**확인 항목**:
- [ ] ✅ 처리 활동: `dpv:Storage` 만 (Transfer 없음)

## 3. 가/익명화 실제 적용 검증 — 핵심

### 시나리오 3.1: 카테고리 A — 평문 직접 교체
**입력**: `test_email.txt` (PII 포함)

**절차**:
1. 분석 → 가명화 "가명처리" 선택 → 봉투 생성
2. **봉투 다시 열어서 파일 내용 확인**

**확인 항목**:
- [ ] ✅ 옵션 단계 패널: "✅ 직접 적용" + 한국어 안내
- [ ] ✅ 봉투 안 파일: PII 가 placeholder 로 교체됨 (`[EMAIL_ADDRESS_1]` 등)
- [ ] ✅ 원본 PII 텍스트가 봉투에 남아 있지 X

### 시나리오 3.2: 카테고리 A — XLSX
**입력**: `test_payroll.xlsx` (셀에 주민번호)

**확인 항목**:
- [ ] ✅ "✅ 직접 적용 (N개 셀의 PII 텍스트가 교체됨)"
- [ ] ✅ 봉투 다시 열기 → 엑셀 셀 내용에서 주민번호 안 보임
- [ ] ✅ 셀 위치·서식 보존

### 시나리오 3.3: 카테고리 A — PPTX / HWPX
**입력**: `test_meeting.pptx`, `test_report.hwpx`

**확인 항목**:
- [ ] ✅ "✅ 직접 적용"
- [ ] ✅ 봉투 다시 열기 → 슬라이드/section 텍스트에서 PII 교체됨
- [ ] ✅ 레이아웃 유지

### 시나리오 3.4: 카테고리 B — DOCX
**입력**: `test_proposal.docx`

**확인 항목**:
- [ ] ✅ "✅ 직접 적용 (N개 DOCX 텍스트 노드 교체됨)"
- [ ] ✅ 봉투 다시 열기 → 문서 텍스트에서 PII 교체됨

### 시나리오 3.5: 카테고리 C — PDF
**입력**: `test_contract.pdf`

**확인 항목**:
- [ ] ✅ "📎 동봉" + sidecar 파일명: `test_contract.pdf.anonymized.txt`
- [ ] ✅ 안내 메시지: "PDF 는 레이아웃 보존 문제로..."
- [ ] ✅ 💡 제안: "DOCX/XLSX/PPTX/HWPX 변환 권장"
- [ ] ✅ 봉투 안에 원본 PDF + sidecar TXT 둘 다 존재
- [ ] ✅ sidecar TXT 안 PII 가 가명화됨

### 시나리오 3.6: 카테고리 C — 이미지
**입력**: `test_namecard.jpg` (명함 사진)

**확인 항목**:
- [ ] ✅ OCR 적용 + "📎 동봉"
- [ ] ✅ sidecar: `test_namecard.jpg.anonymized.txt`
- [ ] ✅ OCR 결과가 가명화되어 sidecar 에 저장됨
- [ ] ✅ 원본 이미지는 그대로

### 시나리오 3.7: HWP 바이너리 + 안내
**입력**: `test_legacy.hwp`

**확인 항목**:
- [ ] ✅ 파일 추가 즉시 ⚠ 토스트: "HWPX 로 저장하시면 가명화가 더 정확히 적용됩니다"
- [ ] ✅ 분석 → 가명화 → "📎 동봉"
- [ ] ✅ 안내: "HWP 바이너리 는 직접 교체 불가"
- [ ] ✅ 💡 제안: "HWPX 로 저장하시면..."

## 4. UI 검증 — 카드 + 검색 + 통계

### 시나리오 4.1: 봉투 카드 표시
**확인 항목**:
- [ ] ✅ 카드 좌측 보조선 색상: high(빨강) / medium(주황) / low(녹색)
- [ ] ✅ DPV 칩이 위험 등급별 색상 (high=빨강 / medium=주황 / low=회색)
- [ ] ✅ hover 시 tooltip 에 전체 카테고리 목록

### 시나리오 4.2: 검색 (DPV 라벨 매칭)
**절차**: 검색창에 "이메일" 입력

**확인 항목**:
- [ ] ✅ `dpv:EmailAddress` 가진 봉투들만 필터링
- [ ] ✅ "주민번호", "여권" 등도 매칭
- [ ] ✅ 영어 라벨 ("Email") 매칭도 동작

### 시나리오 4.3: DPV 카테고리 필터
**절차**: 상단 "모든 DPV" 셀렉트에서 특정 카테고리 선택

**확인 항목**:
- [ ] ✅ 해당 IRI 가진 봉투들만 표시
- [ ] ✅ 한/영 토글 (🇰🇷 KO ↔ 🇬🇧 EN) 동작

## 5. OPA 룰 검증 (DPV 기반)

### 시나리오 5.1: 자격증명 외부 전송 차단
**입력**: AWS_ACCESS_KEY 패턴 포함된 파일 (`AKIA1234567890ABCDEF`)

**절차**:
1. 분석 → 가명화 건너뛰기 → 의도 "외부" 선택

**확인 항목**:
- [ ] ✅ AnalysisDialog 에 "차단" 표시
- [ ] ✅ 메시지: "자격증명이 포함된 봉투의 외부 전송은 차단됩니다 — 즉시 키를 회전(rotate)하세요"

### 시나리오 5.2: 고위험 PII + 외부 + 비가명화 차단
**입력**: 주민번호 포함 텍스트

**절차**: 의도 "외부" + 가명화 "건너뛰기"

**확인 항목**:
- [ ] ✅ "차단" + "고위험 PII (주민번호·여권·신용카드·건강보험·운전면허) 가 포함된 봉투의 외부 전송은 가명/익명화가 필수입니다"
- [ ] ✅ 가명화 적용하면 차단 해제

### 시나리오 5.3: O 등급인데 고위험 PII 검출
**입력**: 비한국어 + 주민번호 포함

**확인 항목**:
- [ ] ✅ 권고 (recommended): "고위험 PII 가 검출됐는데 등급이 O 입니다 — 분류 오류 가능성. 등급 재검토하세요"

## 알려진 한계

다음 케이스는 현재 구현으로 처리 안 되는 것이 정상:
- DOC (구버전 워드, 비 OOXML) — 미지원
- PPT (구버전, 비 OOXML) — 미지원
- HWP 바이너리 — sidecar 동봉만, 직접 교체 X
- PDF — sidecar 동봉만 (레이아웃 보존 문제)
- 이미지 — sidecar 동봉만 (픽셀 텍스트 교체 X)
- 한국어 외 언어 PII (영어 PERSON 등) — 매핑 일부만
